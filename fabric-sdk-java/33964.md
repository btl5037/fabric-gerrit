<strong>Project</strong>: fabric-sdk-java</br><strong>Branch</strong>: release-1.4<br><strong>ID</strong>: 33964<br><strong>Subject</strong>: [FABJ-491] Default grpc/grpcs based on discovery peer<br><strong>Status</strong>: MERGED<br><strong>Owner</strong>: Simon Stone - sstone1@uk.ibm.com<br><strong>Assignee</strong>:<strong>Created</strong>: 10/14/2019, 8:36:58 AM<br><strong>LastUpdated</strong>: 10/15/2019, 6:11:37 AM<br><strong>CommitMessage</strong>:<br><pre>[FABJ-491] Default grpc/grpcs based on discovery peer

Currently grpcs is used for all discovered peers and
orderers. This is a bad assumption to make when the
network is not using TLS. If the discovery peer is
using TLS, then use grpcs, else use grpc. You cannot
have mixed TLS and non-TLS networks, so we don't need
to support discovery peer using TLS and discovered peer
not using TLS.
