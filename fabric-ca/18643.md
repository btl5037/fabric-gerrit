Currently, DB migration logic is run during start up even if the
DB level is same as server level. With this change set, it will
only be run if the DB level is less than server level. Before this
change set logic was:
Update database schema, add users/affiliations from the config file and
migrate the data

After this change set:
Database schema is updated and data is migrated in one transaction and
then users/affiliations defined in the config file are added.
