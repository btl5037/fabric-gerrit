Now in order to know whether a tx existed in ledger (endorse/commit),
the peer calls GetTransactionByID() and ignore the returned
transaction data by checking index and get tx data from file storage.

This way is slow and heavy with performance penalty as the detection
happens at least 2 times for each transaction.

This patchset implements a more lightweighted method by only detecting
the existance of the transaction and no query the real data. In theory,
this will improve the performance a lot by reducing the IO penalty for
every transaction endorsement operation.

On the other hand, we also updated the related testing cases.
