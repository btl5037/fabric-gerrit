This CR initializes the ledger cache for the peer. There are two
caches:
(1) system cache - stores state associated with system chaincodes
    such as lscc.
(2) user cache - stores state associated with user chaincodes.

For each channel, a partition is made in the system cache and
user cache. While creating a channel database or opening an
existing channel database after the peer startup, a partition
is created in the cache for that channel.
