This changeset aims to trigger discussion and finalize the
config information passed to peer and chain (through genesis block)
to setup appropriate MSPs and ACLs on the channel/chain level.

In particular for peer initialization we include here the following:
 - peer-init-config-schema.go includes the schema of the peer
   initialization information. This consists of

   (i) peer MSP related information, i.e., the configuration
    of the MSP the peer's signing identity belongs to, as well
    as the information on this signing identity
   (ii) peer bccsp related information, e.g., location where the
    keys are to be stored, etc.
   (iii) endorsement information for the peer to know the identity
    its ESCCs should use for signing

 - peer-init-config_test.go is a test for the schema defined in
   peer-init-config-schema.go
 - peer-init-config-test-result.json is the output of the test.

For the chain initialization, this changeset includes the following
file:

 - chain-genesis-block-config-schema.go includes the schema for a new
   chain creation.
 - chain-genesis-block-config_test.go is a test to generate a json
   string, with a sample chain cofiguration.
 - chain-genesis-block-test-result.json is the resulting file.

 The chain schema and resulting json file aims to be parsed by the
 orderers and could leverage the framework in fabric/orderer/common/cauthdsl
 to instantiate
  (i)  read policies to define who has read access to chain blocks,
  (ii) write policies to define who has permission to submit
       transactions to the chain,
  (iii)chaincode lifecycle policies, to define who has permission to
       deploy new chaincodes,
  (iv) admin policy, to define who is authorized to make modifications
       to the chain configuration.

This changeset also references:
https://jira.hyperledger.org/browse/FAB-1064
