cryptogen generates the following NodeOUs (when
enabled):
OU=FabricClient
OU=FabricPeer

fabric-ca generates:
OU=client
OU=peer

This causes issues if you try to bootstrap a
network with cryptogen and then enroll clients
with fabric-ca.

It's trivial to make cryptogen use the same
OUs as fabric-ca.
