Chaincode developers have to write the same logic redundantly to
create a composite key before accessing state in the ledger.
To improve the usability, this CR defines the following APIs for
objectType based state access by combining logics to generate
composite keys and to access state in a single function.
The implementation will be another CRs.

(1) GetStateByCompositeKey(objectType string,
     keys []string) ([]byte, error)
(2) PutStateByCompositeKey(objectType string,
     keys []string, value []byte) error
(3) DelStateByCompositeKey(objectType string,
     keys []string) error
(4) SetStateValidationParameterByCompositeKey(objectType string,
     keys []string, ep []byte) error
(5) GetStateValidationParameterByCompositeKey(objectType string,
     keys []string) ([]byte, error)
(6) GetStateByObjectType(objectType string)
    (StateQueryIteratorInterface, error)
(7) GetStateByObjectTypeWithPagination(objectType string,
     pageSize int32, bookmark string)
    (StateQueryIteratorInterface, *pb.QueryResponseMetadata, error)
(8) DelStateByObjectType(objectType string) error
(9) GetHistoryForCompositeKey(objectType string,
      keys []string) (HistoryQueryIteratorInterface, error)
