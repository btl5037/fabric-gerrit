Presently, the chaincode lifecycle only uses the public data to retrieve
the chaincode definition, then reads the package filesystem from disk
(for each invocation).  This was a hack to get the green path going, but
critically, it does not confirm that the org has approved the definition
(and of course the performance will be very poor).  This CR re-wires and
re-codes the information retrieval to use the newly added chaincode
