<strong>Project</strong>: fabric</br><strong>Branch</strong>: release-1.2<br><strong>ID</strong>: 28844<br><strong>Subject</strong>: [FAB-13471] lifecycle to handle multiple updates<br><strong>Status</strong>: MERGED<br><strong>Owner</strong>: Yacov Manevich - yacovm@il.ibm.com<br><strong>Assignee</strong>:<strong>Created</strong>: 1/20/2019, 7:53:17 PM<br><strong>LastUpdated</strong>: 1/22/2019, 11:26:26 AM<br><strong>CommitMessage</strong>:<br><pre>[FAB-13471] lifecycle to handle multiple updates

The ledger calls several HandleChaincodeDeploy for each update
but only a single ChaincodeDeployDone after all invocations
to HandleChaincodeDeploy were made.

The current implementation only supported a single update,
and as a result - a second ChaincodeDeployDone will get stuck
writing to a channel.

This change set makes the Lifecycle in core/cclifecycle
to be able to handle any number of updates in a single block.
