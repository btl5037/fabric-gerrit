<strong>Project</strong>: fabric</br><strong>Branch</strong>: master<br><strong>ID</strong>: 15249<br><strong>Subject</strong>: [FAB-6803] Rename capabilities key V1.1 to V1_1<br><strong>Status</strong>: MERGED<br><strong>Owner</strong>: Jay Guo - guojiannan1101@gmail.com<br><strong>Assignee</strong>:<strong>Created</strong>: 11/7/2017, 11:00:12 AM<br><strong>LastUpdated</strong>: 11/8/2017, 9:17:26 AM<br><strong>CommitMessage</strong>:<br><pre>[FAB-6803] Rename capabilities key V1.1 to V1_1

We are using an old version of Viper, which has problems locating
keys with period. It uses period as delimiter for the path of a
nested structure, therefore, foo.bar.V1.1 would result in
foo:
  bar:
    V1:
      1
instead of
foo:
  bar:
    V1.1:
as we expected. This patch simply renames "V1.1" to "V1_1" to avoid
this problem.
