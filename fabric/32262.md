Currently, the run-unit-test.sh script runs in 3 phases, first it tests
all Fabric packages or those specified by TEST_PKGS, then it tests the
packages which must be built with the plugins go tag, then the packages
which must be built with the pkcs11 go tag.  The latter two are
presently not filtered to be a subset of the specified TEST_PKGS.

This CR simply applies the same logic used to filter the serial/parallel
tests to pick the subset of the specified packages which require the
plugins or pkcs11 tags.
