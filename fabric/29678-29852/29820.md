This task focuses on:
- testing the migration "abort" path
- testing the migration crash recovery mechanisms

The main contribution is in adding several test cases to migration_test.go.
The two test-cases that check the abort path are:

1. A test that executes the migration abort-path flow on the Kafka side
   (from START-TX, via ABORT-TX until another START-TX), on the system
   channel only.

2. A test that executes the migration abort-path flow on the Kafka side
   (from START-TX, CONTEXT-TX via ABORT-TX until another START-TX), on
   the system channel and a single application channel.

The test-cases that check migration crash recovery are:

1. Checking that a single node can stop and restart in the middle of
   migration - no catch up from Kafka.

2. Starting 3 orderers, stopping one, continue migration with other 2.
   Restarting the stopped node and checking that it can restore its
   state and catch up with migration from Kafka.
