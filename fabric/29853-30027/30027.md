After a rollback, we need to quickly get the peer to
a pre-rollbacked block height by reprocessing blocks.

During this period, we can disable the
reconciler to avoid interference with the block
reprocessing (would help in reducing the time taken
to reprocess blocks). Further, the missing pvtData
info in the pvtdataStore could belong to a block
which is yet to be processed. In this case, we cannot
run reconciler and try to update the stateDB. Otherwise,
it would end up in a inconsistent stateDB.

The missing pvtData reconcilation is disabled by
returning nil missingDataInfo.
