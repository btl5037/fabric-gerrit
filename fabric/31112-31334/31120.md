External chaincode can be treated as a new type of chaincode platform. This
fits with not only with the existing runtime model, it also fits with the
new lifecycle deployment model where the chaincode "package" (and the
package identifier) is separated from the chaincode "name". The former is
part of the deployment model while the later part of the transaction (via
invoke) model. In particular treating external chaincode as a new platform
allows us to fit deployment of the external chaincode with the chaincode
framework without altering the invoke interactions.
