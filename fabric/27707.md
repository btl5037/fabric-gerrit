<strong>Project</strong>: fabric<br><strong>Branch</strong>: master<br><strong>ID</strong>: 27707<br><strong>Subject</strong>: Prevent verbose error reporting from grpc logging<br><strong>Status</strong>: MERGED<br><strong>Owner</strong>: Matthew Sykes - sykesmat@us.ibm.com<br><strong>Assignee</strong>:<br><strong>Created</strong>: 11/22/2018, 9:21:42 AM<br><strong>LastUpdated</strong>: 11/27/2018, 2:16:01 PM<br><strong>CommitMessage</strong>:<br><pre>Prevent verbose error reporting from grpc logging

By default, for errors that implement fmt.Formatter, zap will add a
verboseError field that includes the '%+v' formatted error in the log
record. This is really cool and useful but it can be noisy.

In order to prevent this noise in the grpc interceptors, we can embed the
errors we're logging in a struct; the result is that the struct
satisfies the error interface without implementing fmt.Formatter.
