<strong>Project</strong>: fabric</br><strong>Branch</strong>: master<br><strong>ID</strong>: 30395<br><strong>Subject</strong>: [FAB-14770] sendReceive is synchronous<br><strong>Status</strong>: MERGED<br><strong>Owner</strong>: Matthew Sykes - sykesmat@us.ibm.com<br><strong>Assignee</strong>:<strong>Created</strong>: 3/25/2019, 6:14:19 PM<br><strong>LastUpdated</strong>: 3/28/2019, 9:54:27 AM<br><strong>CommitMessage</strong>:<br><pre>[FAB-14770] sendReceive is synchronous

The implementation would send a message in a go routine and wait for the
send to complete concurrent with the wait for a response. That pattern
is unnecessary though, as the response will never come if the message
was not sent successfully.

Simplify by moving to serialSend and handling the error before moving on
to wait for the response.
