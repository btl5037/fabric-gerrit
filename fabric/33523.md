<strong>Project</strong>: fabric</br><strong>Branch</strong>: master<br><strong>ID</strong>: 33523<br><strong>Subject</strong>: FAB-16605 Add log message for slow WAL<br><strong>Status</strong>: MERGED<br><strong>Owner</strong>: Jason Yellick - jyellick@us.ibm.com<br><strong>Assignee</strong>:<strong>Created</strong>: 9/13/2019, 4:34:06 PM<br><strong>LastUpdated</strong>: 9/16/2019, 5:08:23 PM<br><strong>CommitMessage</strong>:<br><pre>FAB-16605 Add log message for slow WAL

When calls to store data in the WAL take too long, bad things will
happen in the network.  Notably, if the leader takes close to, or more
than the election timeout value, then the network will elect a new
leader and end up discarding the transaction he's working on.

This CR adds a simple log warning statement, which is printed if the WAL
storage time exceeds half of the configured election timeout.
