ACLs for channeless operations such as LSCC's install or CSCC's
JoinChain use low level ACL APIs directly. This change paves way for using
aclmgmt's "CheckACL" API's uniformly. The ACL support is fixed and provided
out of the defaultprovider but in future we could consider having users
specify policies for these resources.

Note that the call to CheckACL with empty channelID conveniently resolves to
call to policy.CheckPolicyNoChannel (same call used for channeless ACL calls
in the system chaincodes).
