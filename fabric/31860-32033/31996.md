Historically the shim was included in the ccenv image
as a convenience to prevent having to vendor the shim
for every chaincode application written. Go dependency
management model has never fully matured and there remain
edge cases where the vendored version of packages conflicts
with the fabric version included in ccenv. The shim will be
removed from ccenv and the version vendored locally will
be copied from the $GOPATH into the chaincode package
