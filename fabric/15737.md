Error code is not returned because response value is ignored.
- ProcessProposal function in core/endorser/endorser.go returns a
  response and an error value if an error happens.
- After that, a response and an error value are passed to
  ProcessProposal function in protos/peer/peer.bp.go.
- This function only returns the error value. Even though response
  value is set, the response value is ignored.

This CRs fixes this problem by changing returned value.
- ProposalResponse function returns only response value even if an
  error happens.
- We can get error code because an error value is included in
  response value.
