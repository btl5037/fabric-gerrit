In this commit, we:

1. Try to serve vdb.GetState() from cache, or store the value from DB
   into the cache if there was a miss.
2. Modify ApplyUpdates' revision collection to occur from the stateCache.
3. Update versions, metadata and (optionally) values after committing.

We don't always store values, since caching values always might be burdensome
for many potential deployment scenarios. So it'd be beneficial to store them
only in case GetState was called on those values. We store the version and
metadata nevertheless as it'd be likely useful in future block commits.
