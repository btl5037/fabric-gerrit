This change set introduces the necessary changes to support an LSCC
deploy/upgrade request when the channel operates with the LifecycleViaConfig
capability. In particular, the code checks that the chaincode that is being
deployed/upgraded is already known in the config tree and that data on the
file system and data on the config tree match (i.e. that version and hash
are the same). If so, the code proceeds to create an entry for the chaincode
in LSCC's namespace as before.
