For pagination, chaincode needs to pass a page size and a
bookmark when performing a range/rich query. As changing
existing range/rich query would break the backward compatibility
of existing chaincodes/prod-system, this CR introduces the following
chaincode APIs.
(1) GetStateByRangeWithPagination(startKey, endKey, pageSize, bookmark)
(2) GetQueryResultWithPagination(query, pageSize, bookmark)
(3) GetStateByPartialCompositeKeyWithPagination(objectType, keys,
				pageSize, bookmark)
