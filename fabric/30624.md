in the pvtdataStore, we do store missing info for both
valid and invalid tx. When the reconciler fetches the
missing pvtData, it passes to the ledger coordinator.
As the pvtData contains data of both valid and invalid tx,
we need to pass both to pvtdataStore and only valid tx's
pvtData to the txmgr.

Similarly, during failure and recovery, we fetch the pvtData
of last updated old blocks and filter uncommitted tx pvtData
while syncing the stateDB
