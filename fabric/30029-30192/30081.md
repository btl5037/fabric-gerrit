Currently, the ccenv image is used to build Node.js CC,
whilst the nodeenv image is used to run Node.js CC. This
is not ideal as it means the Node.js version must be in
sync across both images.

For Java CC, the javaenv image is used to build + run
Java CC. We should do the same for Node.js.
